Changelog:
1.  `todims` now refactor as `tobase` to better indicate that the function converts to base units (like `ubase`)
2.  New LogQuant type supported with its own algebra rules (based on logarithmic identities)
3.  log(q<:Quantity) now produces a LogQuant instead of an error
4.  Logarithmic unit functions like `dB` and `Np` are available (but not exported by default to prevent namespace pollution)

1.  Add interactive unit functionality
    -   UPREFERRED is a dict with dimension keys and preferred simplification units
        -   Lives outside the unit registry, and is updated by the default unit registry becuse it has to be Affine units
        -   Uses SI units for dimensions by default (N, J, W, V, Ohms)
        -   May need brackets around compound dimensiosn like velocity = Symbol("(m/s)")
        -   Units cannot have an offset
        -   Does not update if an inverse dimension already exists ('Hz' won't be added if 's' exists)
            -   This is because greedy search includes inverses of units, if both units and inverse exist, results are not unique
            -   Failed attempts must produce a warning, but building defaults can suppress this warning
    -   simplify(UPREFERRED, q::Quantity{T,AbstractDimLike}) 
        -   Only works if all unit powers are integers
        -   Iterates through UPREFERRED, performs a greedy search to make units as terse as possible
            -   e.g. simplify(1.0u"(m^2*kg)/s^3") = 1 kW
            -   breakdown = Dict{Dimensions, Int64}()
            -   For query unit (u), use the "fits into" criterion for selection 
                -   Define abs(d::Dimensions) = sum(abs, d)
                -   If dividing by the unit reduces its absolute value by the same absolute value as (u), it fits cleanly
                    -   Both the unit and the inverse are tested
                        -   (abs(dimension(q))) - abs(dimension(q)/dimension(u)) == abs(u)
                            -   breakdown[dimensions(u)] = get(breakdown, dimensions(u), D()) + 1
                        -   (abs(dimension(q))) - abs(dimension(q)*dimension(u)) == abs(u)
                            -   breakdown[dimensions(u)] = get(breakdown, dimensions(u), D()) - 1
                -   In order to match whole powers (if a unit fits more than once) iterate until it no longer fits
    -   Can add a global option switch for "display" to turn simplification on (it's off by default)
    -   simplify(tempunits::AbstractVector{Units}, q::Quantity{T,AbstractDimLike})
        -   Modifies a copy of UPREFERRED to prevent overwriting defaults

2.  Add plot recipes

import DynamicQuantities
import Unitful
using BenchmarkTools

v1flex = ubase.([1u"m/s", 1u"J/kg", 1u"A/V"])
v1uni  = [1*Unitful.u"m/s", 1*Unitful.u"J/kg", 1*Unitful.u"A/V"]
v1dyn  = [1*DynamicQuantities.u"m/s", 1*DynamicQuantities.u"J/kg", 1*DynamicQuantities.u"A/V"]

@btime sum(x->x^0.0, v1uni)
@btime sum(x->x^0.0, v1flex)
@btime sum(x->x^0.0, v1dyn)

t1flex = ubase.((1u"m/s", 1u"J/kg", 1u"A/V"))
t1uni  = (1*Unitful.u"m/s", 1*Unitful.u"J/kg", 1*Unitful.u"A/V")
t1dyn  = (1*DynamicQuantities.u"m/s", 1*DynamicQuantities.u"J/kg", 1*DynamicQuantities.u"A/V")

@btime sum(x->x^0, t1uni)
@btime sum(x->x^0, t1flex)
@btime sum(x->x^0, t1dyn)
